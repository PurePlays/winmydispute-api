🧠 Expanded Merchant Defense Tactics to Anticipate

✅ Non-Delivery Disputes:

- Merchant provides fraudulent tracking data
- Merchant claims third-party shipper responsibility
- Merchant claims pickup without user signature
- Merchant claims digital good "accessed" based only on internal server logs
- Merchant claims delays "don’t qualify" as non-delivery

✅ Defective/Not as Described Disputes:

- Merchant claims "minor variation" isn't misrepresentation
- Merchant claims damage after delivery
- Merchant uses vague listing terms to argue user misunderstood
- Merchant refuses return but blames user
- Merchant provides biased internal inspections

✅ Fraud/Unauthorized Transaction Disputes:

- Merchant claims user authenticated via 3D Secure
- Merchant highlights prior transaction history to argue implied consent
- Merchant shows device/IP match
- Merchant uses internal fraud reports favoring themselves

✅ Cancellation/Refund Promised Disputes:

- Merchant claims refund "in process" indefinitely
- Merchant blames user's bank for missing refund
- Merchant hides behind obscure non-refundable clauses
- Merchant denies cancellation after "short windows"

✅ Subscription/Auto-Renewal Disputes:

- Merchant claims auto-renew was disclosed (buried small print)
- Merchant sends minimal or no notification before renewal
- Merchant claims user agreed simply by account creation
- Merchant uses confusing billing descriptors to delay recognition

✅ Rare/Dirty Tactics:

- Fabricated "chat logs" or "emails"
- Fake delivery screenshots
- Blame shifting to payment processors
- Ignoring user entirely hoping deadlines expire
- Mislabeling transaction type deliberately

✅ REMINDER:  
**You must dynamically anticipate based on reason code, user facts, and merchant type.**  
**Always assume rebuttals will happen, and prepare to crush them first.**

🧠 Full Preemptive Counter-Strategy Playbook (Dynamic)

✅ Non-Delivery Preemptive Moves:

- Independent tracking screenshots (carrier portal screenshots, not merchant links)
- GPS delivery map overlays
- Carrier claims reports if package lost or misdelivered
- Lack of signed delivery confirmation emphasized
- Affidavits of non-receipt if needed

✅ Defective/Not As Described Preemptive Moves:

- Side-by-side photos comparing listing vs received item
- Photo metadata (timestamp proofs)
- Third-party inspection letters
- Merchant response (or lack thereof) to complaints documented
- Highlight ambiguous or misleading listing descriptions

✅ Fraud/Unauthorized Preemptive Moves:

- Affidavit of unauthorized use
- Proof of non-matching devices or IP addresses
- Highlighting fraud alerts, prior password reset warnings, card reissuance if occurred

✅ Cancellation/Refund Preemptive Moves:

- Screenshots of cancellation submissions
- Email chains requesting refund
- Proof of no refunds after reasonable timeframes
- Clear user expectation of refund (based on merchant communications)

✅ Subscription Dispute Preemptive Moves:

- Lack of clear opt-in records
- Screenshots of confusing signup screens
- Absence of pre-renewal notifications
- Immediate cancellation request evidence

✅ Dirty Tactics Defense Moves:

- Lack of independent communication proof
- Questionable internal-only screenshots flagged
- Highlighting merchant delay as bad faith under FTC unfair practices

✅ Always Tie Preemptive Moves Back to:
- Fair Credit Billing Act (for cardholder protections)
- Card Network Chargeback Rules (Visa, Mastercard, Amex, Discover)
